
2. Pengujian API FO (Fiber Optic)

---

a. Login Superadmin
Untuk menguji proses login, permintaan dikirim ke endpoint `/superadmin/auth/login` menggunakan metode POST dengan menyertakan email dan password. Sistem merespons dengan token autentikasi dan data pengguna, menandakan login berhasil. Hasil pengujian ini dapat dilihat pada Gambar 5.0.

b. Statistik/FTTH Report
Pengambilan data statistik FTTH dilakukan dengan mengirim permintaan GET ke endpoint `/api/v1/fo-client-ftths/statistik`. Sistem memberikan data statistik yang lengkap, menandakan fitur ini berjalan sesuai harapan. Dokumentasi hasil pengujian tersedia pada Gambar 5.1.

---

### FoLokasi

1. Create
Penambahan data lokasi dilakukan dengan mengirim permintaan POST ke `/api/v1/fo-lokasis` beserta data nama_lokasi, deskripsi, latitude, longitude, dan status. Sistem mengembalikan data lokasi baru dan status sukses, menandakan proses berjalan lancar. Ilustrasi pengujian dapat dilihat pada Gambar 5.2.

2. List
Untuk melihat daftar lokasi, permintaan GET dikirim ke `/api/v1/fo-lokasis`. Sistem menampilkan seluruh data lokasi yang tersimpan, menandakan fitur list berfungsi baik. Hasil pengujian ini dapat dilihat pada Gambar 5.3.

3. Update
Proses pembaruan data lokasi dilakukan melalui endpoint `/api/v1/fo-lokasis/{id}` dengan metode PUT/PATCH dan data yang ingin diubah. Sistem mengembalikan data terbaru, menandakan update berhasil. Dokumentasi pengujian tersedia pada Gambar 5.4.

4. Delete
Penghapusan lokasi dilakukan dengan permintaan DELETE ke `/api/v1/fo-lokasis/{id}`. Sistem menandai data sebagai terhapus (soft delete) dan mengirim respons sukses. Hasil pengujian dapat dilihat pada Gambar 5.5.

5. Archive
Untuk mengarsipkan lokasi, PATCH dikirim ke `/api/v1/fo-lokasis/{id}/archive`. Sistem mengubah status menjadi archived dan mengonfirmasi keberhasilan. Simulasi pengujian ini dapat dilihat pada Gambar 5.6.

6. Unarchive
Pengaktifan kembali lokasi yang diarsipkan dilakukan dengan PATCH ke `/api/v1/fo-lokasis/{id}/unarchive`. Sistem mengembalikan status menjadi active. Dokumentasi pengujian tersedia pada Gambar 5.7.

7. Restore
Pemulihan data lokasi yang terhapus dilakukan dengan PATCH ke `/api/v1/fo-lokasis/{id}/restore`. Sistem mengembalikan data ke kondisi aktif. Hasil pengujian dapat dilihat pada Gambar 5.8.

8. Bulk
Operasi bulk (archive, delete, restore) dilakukan dengan POST ke `/api/v1/fo-lokasis/bulk` beserta daftar ID dan aksi. Sistem memproses setiap permintaan secara massal dan mengirimkan status sukses. Ilustrasi pengujian dapat dilihat pada Gambar 5.9.
